üö® 1. Error Handling in FastAPI

‚úÖ Built-in Exception Handling

FastAPI provides built-in exceptions like HTTPException.

from fastapi.exceptions import RequestValidationError

        status_code=422,
        content={"message": "Validation failed üòµ", "details": exc.errors()},

üîê Best Practices

  • Use descriptive detail in errors.
  • Handle common errors (e.g., 404, 400, 500) globally.
  • Avoid leaking sensitive info in error messages.

üåê HTTP Status Codes Cheat Sheet with FastAPI üí°

üîµ 1xx ‚Äî INFORMATIONAL RESPONSES

‚úÖ Request received, continuing process.

Code Meaning Description
100 Continue Client should continue the request.
101 Switching Protocols Server agrees to change protocols.

üß† FastAPI Tip: Rarely used directly.

üü¢ 2xx ‚Äî SUCCESSFUL RESPONSES

üéâ The request was successfully received, understood, and accepted.

Code Meaning Description
200 OK Standard success for GET/PUT/DELETE.
201 Created Used after successful POST (e.g., resource created).
202 Accepted Request accepted but not completed yet.
204 No Content Success, but no response body (e.g., DELETE).

üì¶ FastAPI Usage Example:

    return {"message": "User created ‚úÖ"}

üü° 3xx ‚Äî REDIRECTION MESSAGES

üîÅ The client must take additional action.

Code Meaning Description
301 Moved Permanently Resource has a new URL. Update your links!
302 Found (Temporary) Temporarily moved to another URI.
304 Not Modified Cached version can be used.
307 Temporary Redirect Like 302 but preserves method (POST/GET).
308 Permanent Redirect Like 301 but with method preservation.

üì¶ FastAPI Example:

@app.get("/old-link")

    return RedirectResponse(url="/new-link", status_code=307)

üî¥ 4xx ‚Äî CLIENT ERROR RESPONSES

üö´ The request contains bad syntax or cannot be fulfilled.

Code Meaning Description
400 Bad Request Client error (validation, format, etc).
401 Unauthorized Authentication required.
403 Forbidden Authenticated but not allowed.
404 Not Found Resource doesn‚Äôt exist.
405 Method Not Allowed Method (GET/POST/etc) not supported.
409 Conflict Resource conflict (duplicate user, etc).
422 Unprocessable Entity Validation failed (e.g., FastAPI Pydantic errors).
429 Too Many Requests Rate limit hit.

üì¶ FastAPI Example (422 Validation):

üí• The server failed to fulfill a valid request.

Code Meaning Description

üö® FastAPI ‚Äî Exceptions & Error Handling

üîπ What Are Exceptions?

Exceptions in FastAPI allow us to notify the client that something went wrong in a clean and structured way.

üìå When you raise an exception, it stops the execution of your code at that point and returns a well-defined error response to the client.

üß± Built-in Exception ‚Äî HTTPException

FastAPI provides a built-in exception:

from fastapi import HTTPException

üßë‚Äçüíª Example: Raise a 404 Not Found Error

from fastapi import FastAPI, HTTPException

app = FastAPI()

fake_users_db = {"1": "Alice", "2": "Bob"}

@app.get("/user/{user_id}")
def get_user(user_id: str):
    user = fake_users_db.get(user_id)
    if not user:
        # üö® Raise an HTTP Exception
        raise HTTPException(
            status_code=404,  # Not Found ‚ùå
            detail=f"User with id {user_id} not found"
        )
    return {"user": user}

üîç Output If User Not Found:

{
  "detail": "User with id 6 not found"
}

üß† Explanation:

  • status_code=404: Tells the client this is a "Not Found" error.
  • detail: Gives human-readable error info.
  • ‚úÖ FastAPI automatically structures this as a proper JSON error response.

üî¢ Common HTTP Status Codes

Code Name Use Case
400 Bad Request Invalid input, wrong format üö´
401 Unauthorized Missing or bad auth header üîê
403 Forbidden Valid credentials, but access denied üö∑
404 Not Found Resource doesn't exist üîé
409 Conflict Duplicate resource or version clash ‚öîÔ∏è
422 Unprocessable Entity Validation errors (FastAPI default) ‚ùå

üéØ Best Practice: Raise Early

‚úÖ Raising exceptions early prevents further code from executing. Example:

    raise HTTPException(status_code=400, detail="Invalid input data üß®")
# ‚õî this line won't run if exception is raised

üîÄ 8. CORS (Cross-Origin Resource Sharing)

üí° Allows your backend (e.g., localhost:8000) to talk to a frontend on another domain (e.g., localhost:3000)

Enable CORS

from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000"],  # frontend
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

üß† Why?

Without this, browsers block cross-origin AJAX calls due to same-origin policy.
